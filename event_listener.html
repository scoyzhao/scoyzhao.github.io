冒泡与捕获

监听事件需要关注的两个概念，就是冒泡与捕获，二者是网景公司和微软解决事件流提出的两种完全不同的解决方案：

其中，1-5是捕获，5-6是目标，6-10是冒泡

addEventListener

从接口来看，主要是在capture相关的来区分在哪个阶段捕获事件

值得注意的是，同一个函数，绑定冒泡阶段和捕获阶段会触发两次，是不同的

场景

这里想到了一个场景，那就是ul代理li的事件，这样的好处是，无论li有几个，只需要在ul加事件，冒泡上去就可以了

阻止事件冒泡

  1. 给子级加event.stopPropagation()
  2. 处理事件函数返回false

是这两种方式是有区别的。return false不仅阻止了事件往上冒泡，而且阻止了事件本身(默认事件)。event.stopPropagation()则只阻止事件往上冒泡，不阻止事件本身。

  1. event.target = event.currentTarget

阻止默认事件

  1. event.preventDefault()
  2. return false

addEventListener与removeEventListener

一般来说，注册listener，需要一个listener，remove的时候要用同一个函数才可以解绑。这就必须得记录下这个listener，这给编程带来了一些麻烦，新增这个变量

addEventListener提供了另外一种处理监听事件的方法，那就是用 handleEvent 方法：

demo：

var Something = function(element) {
  // |this| is a newly created object
  this.name = 'Something Good';
  this.handleEvent = function(event) {
    console.log(this.name); // 'Something Good', as this is bound to newly created object
    switch(event.type) {
      case 'click':
        // some code here...
        break;
      case 'dblclick':
        // some code here...
        break;
    }
  };

  // Note that the listeners in this case are |this|, not this.handleEvent
  element.addEventListener('click', this, false);
  element.addEventListener('dblclick', this, false);

  // You can properly remove the listeners
  element.removeEventListener('click', this, false);
  element.removeEventListener('dblclick', this, false);
}
var s = new Something(document.body);
