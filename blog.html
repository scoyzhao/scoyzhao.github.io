1.背景

我已经做过多个blog，但是由于自己服务器没有维护也好，或者太麻烦也好，也都荒废了。今年开始笔记软件也从为知笔记前移到了notion，于是就突发奇想，有没有方法把notion的笔记当做blog，这样就可以直接把notion中的一些自己写的东西分享出来，也不需要特意去维护blog。

通过一些搜索，发现了一些方案，基本都是通过github的静态页面来部署，最终选择了一个比较方便的，样式简单，管理方便的案例，也遇到了一些问题，特此记录。

参考方案写在最前：https://zhuanlan.zhihu.com/p/469320294

创建access token

这一步目的是给这个项目权限去把解析完的notion内容部署到我们的仓库

2.给部署仓库

回到你在 步骤4中fork的名为 notablog-starter仓库，依次点击 settings→ secrets→ actions→ New repository secre， Name 填为 ACCESS_TOKEN，Value 就是上一步生成的字符串

3.部署仓库添加actions

这一步是为了让他部署/自动每天部署

回到你在的名为 notablog-starter仓库，依次点击 Actions→ New workflow→ set up a workflow yourself

写入下列代码，注意仓库和分支要修改

# This is a basic workflow to help you get started with Actions
    
    name: Github Pages
    
    # Controls when the workflow will run
    on:
      schedule:
    		# 下一行是说在每天的国际标准时间23点（北京时间早上7点）触发该任务
    		# 你可以随意修改
        - cron: '0 23 * * *'    
      # Allows you to run this workflow manually from the Actions tab
      workflow_dispatch:
    
    # A workflow run is made up of one or more jobs that can run sequentially or in parallel
    jobs:
      # This workflow contains a single job called "build"
      build:
        # The type of runner that the job will run on
        runs-on: ubuntu-latest
    
        # Steps represent a sequence of tasks that will be executed as part of the job
        steps:
          # 拉取代码
          - name: Checkout
            uses: actions/checkout@v2
          # 1、生成静态文件
          - name: Build
            run: npm i -g notablog && notablog generate .
          # 2、部署到 GitHub Pages
          - name: Deploy
            uses: JamesIves/github-pages-deploy-action@v4.2.5
            with:
              token: ${{ secrets.ACCESS_TOKEN }}
              repository-name: FizzerYu/FizzerYu.github.io # 修改这里
              BRANCH: main  # 如果你的仓库默认分支是 master 记得修改这里                    
              FOLDER: public

然后就可以通过action来部署啦
